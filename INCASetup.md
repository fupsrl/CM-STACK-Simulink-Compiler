# INCA Setup

A simple simulink model was built just for example
<img width="1270" height="474" alt="image" src="https://github.com/user-attachments/assets/891d608f-1905-4a37-8433-48ff5b04fdc1" />


After building and deploying your model, you should see this structure into your main folder:

Main Folder>workflow>artifacts>testSimulinkRaspberry.a2l                  : vanilla a2l generated by XCPLite (valid for CANAPE)
Main Folder>workflow>artifacts>testSimulinkRaspberry.hex                  : hex from compiled elf binary
Main Folder>workflow>artifacts>testSimulinkRaspberry.s19                  : s19 from compiled elf binary
Main Folder>workflow>artifacts>testSimulinkRaspberry_inca.a2l             : patched a2l for INCA
Main Folder>workflow>artifacts>testSimulinkRaspberry_raw.hex              : raw hexfile
Main Folder>workflow>artifacts>XCP_104.aml                                : vanilla XCP protocol layer by XCPLite


## INCA Import

Prerequisite: The Raspberry Pi must be set to have a static IP address.
Up to now, raspberry pi os Lite (trixie) has no static IP by default assigned.
